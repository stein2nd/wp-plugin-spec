# SPEC_DATA_DICTIONARY.md (データ、設定値、ストレージ定義)

> 各プラグインリポジトリ直下の `docs/` フォルダ等にコピーして利用することを想定しています。
> ファイル名の一例: `docs/SPEC_DATA_DICTIONARY.md`

* **目的**: Options API、Post Meta、Custom Table など、**どこに何を保存するか** をカラムレベルで定義する。
* **内容例**:
  * `option_name` / `meta_key` / テーブルカラム名
  * 型、nullable、デフォルト値
  * どの画面や API から読み書きされるか
* **AI 的メリット**:
  * データ移行やリファクタリング時に、AI に **スキーマを誤解させない**。
  * TypeScript の型定義と1対1で対応させやすい。

## 4. AI 伴走開発での使い方メモ

* **データ移行・リファクタリング**:
  * 本ファイルを AI に渡し、「option_name / meta_key を変更するが、既存データの移行コードも提案してほしい」のように依頼する。
* **型定義の同期**:
  * TypeScript の型定義 (`src/types/` 等) と1対1対応させたい場合は、本データ辞書を参照させてから型を生成・更新するよう依頼する。
* 参照させる関連仕様:
  * `SPEC_UI_AND_FLOWS.md` (どの画面で読み書きするか)、`SPEC_API_AND_INTEGRATION.md` (どのエンドポイントで読み書きするか)。
