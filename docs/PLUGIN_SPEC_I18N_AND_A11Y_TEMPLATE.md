# SPEC_I18N_AND_A11Y.md (国際化/アクセシビリティ)

> 各プラグインリポジトリ直下の `docs/` フォルダ等にコピーして利用することを想定しています。
> ファイル名の一例: `docs/SPEC_I18N_AND_A11Y.md`

* **目的**:
  * 「国際化」「アクセシビリティ」を、プラグイン単位でどう守るかを明記する。
* **内容例**:
  * Text Domain、翻訳対象となる文字列の方針
  * RTL 対応の方針
  * キーボード操作や ARIA 属性の最低ライン
* **AI 的メリット**:
  * 文言生成や UI 実装を AI に任せたときに、「i18n/A11y の抜け」をレビューしやすくなる。

## 1. i18n (国際化) 基本方針

* 本章では、「多言語対応」を記載します。

* テキストはすべて `__()` / `_e()` / `_x()` / `_n()` を適切に使用する (複数形対応を含む)。日付・数値フォーマットについては、地域別の表示形式での対応も考慮する。「RTL 言語」対応も考慮する。

### 1.1. Text Domain / slug

* plugin-slug: フォルダー名/本体 php/テキストドメイン
* Text Domain: plugin-slug に合わせる。
* 翻訳ファイル格納先: `languages/`

### 1.2. 翻訳対象 (UI 文言) のルール

* PHP:
  * `__()`, `_e()`, `_x()`, `_n()` を適切に使う (文脈が必要なら `_x()`、複数形は `_n()`)。
  * 文字列連結で文章を作らない (翻訳が壊れるため)。`sprintf()` とプレースホルダを使う。
* JS/TS (Gutenberg / 管理画面 UI):
  * `@wordpress/i18n` の `__()`, `_x()`, `_n()`, `sprintf()` を使用する。
  * React 側の文言は「直書き禁止」を基本とし、i18n 経由に統一する。

### 1.3. 日付・数値・通貨

* 表示形式はロケール依存を考慮する。
* 依存 API (例):
  * PHP: `wp_date()`, `number_format_i18n()`
  * JS: `Intl.DateTimeFormat`, `Intl.NumberFormat` (必要に応じて)

### 1.4. RTL (右から左) 対応方針

* 管理画面 UI の CSS で、方向依存の指定を避ける (`margin-left` 等の固定ではなく logical properties を検討)。
* ブロックのフロント CSS でも、方向依存の表現を避ける。

## 3. A11y (アクセシビリティ) 基本方針

* **スクリーン・リーダー**:
  * 音声読み上げに対応する。

### 3.1. 最低ライン (必須)

* **キーボード操作**:
  * 主要 UI (保存、追加、削除、モーダル操作等) が、キーボードのみで完結する。
  * フォーカス移動が、破綻しない (Tab 移動、モーダル内フォーカストラップ等)。
* **ARIA**:
  * ボタンは、`button` を使い、`div` クリックで代用しない。
  * ラベル (`aria-label` / `aria-labelledby`) を適切に設定する。
  * 状態 (`aria-expanded`, `aria-busy`, `aria-invalid` 等) を必要に応じて設定する。
* **フォーム**:
  * エラーは、視覚だけでなくテキストで説明し、エラーメッセージへ関連付ける。
  * 必須項目は、「必須」であることが分かる表現を統一する。
* **コントラスト**:
  * WCAG の2.1(AA) を目標にする (特に文字と背景)。

### 3.2. Gutenberg / WP 管理画面 UI との整合

* 可能な限り WordPress の UI コンポーネント (`@wordpress/components`) を利用し、アクセシビリティのベースラインをそろえる。

## 4. チェックリスト (レビュー観点)

### 4.1. i18n

* [ ] すべての表示文言が i18n 関数を経由している
* [ ] 文章の連結がなく、プレースホルダで構築している
* [ ] 文脈が必要な文言は `_x()` を使っている

### 4.2. A11y

* [ ] キーボード操作で主要機能が利用できる
* [ ] `aria-*` が適切で、ラベルが欠落していない
* [ ] エラーメッセージが入力と関連付けられている
